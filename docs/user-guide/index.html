Using DataForms

DataForms lets you build registers — structured, typed tables — and work with their records right in Nextcloud, no spreadsheets and no code. This guide walks through the records table, the workspace you'll use every day, and everything around it.

Key concepts

A handful of words show up everywhere in DataForms. Learn these five and the rest follows.

TermWhat it means
RegisterA table with a defined shape — like a sheet, but every column has a type. Equipment Inventory is a register.
FieldA column in a register, with a type (text, date, select, user, relation…). Status and Due back are fields.
RecordOne row — a single item. “Canon EOS R6, asset EQ-0042” is a record.
FormA data-entry screen for adding/editing a record, optionally split into sections.
ViewA saved combination of columns + filter + sort + search you can return to (and share).
RuleLive form logic — show/hide a field, require it, set a default, validate, or compute it.
AutomationA server-side action that runs when a record changes (notify, set a field, create a folder…).
Roles, in one line

Managers design the register (fields, forms, rules, automations, sharing). Writers add and edit records. Readers browse. What you can do depends on how the register is shared with you.

Our example register

Throughout this guide we use an Equipment Inventory — a register for tracking gear that staff borrow and return (cameras, laptops, lab kit). It links to a small Categories register so each item has a tidy category.

Fields in “Equipment Inventory”

Inventory # · Auto Asset tag · Text · unique Name · Text Category · Relation → Categories Status · Select Condition · Select Assigned to · User Due back · Date Insured · Yes/No Purchase date · Date Notes · Long text

Status options: Available, On loan, In maintenance, Retired. Condition: New, Good, Fair, Poor.

Opening a register

  1. Open DataForms from the Nextcloud app menu at the top of the screen.
  2. Pick Equipment Inventory from the left-hand navigation, or its card on the dashboard.
  3. The register opens on its Records tab — the table. That's where the rest of this guide lives.

The records table

The table is the heart of DataForms. Everything you do with data — find it, sort it, fix a typo, open a row, add a new one — happens here.

Anatomy of the table

Across the top sits the toolbar; below it, your records, one per row. The header row stays pinned as you scroll, and the row-actions (open / edit / delete) stick to the right edge on wide tables.

Toolbar itemWhat it does
SearchFree-text search across the register's text fields.
Saved viewSwitch between saved views (appears once you have any).
FilterBuild precise conditions (Status is On loan, Due back before today…).
New recordAdd an item — blank or via a form.
⋯ MoreRefresh, save/delete a view, import/export CSV, and the Columns picker.

Searching

Type in the Search box to narrow the table to records whose text matches — for example Canon finds every Canon item. Search combines with any active filter.

Filtering

Search is broad; filters are precise. Open Filter and add one or more conditions — each is a field · operator · value row. Conditions combine with AND (all must match).

OperatorsFor
is / is notExact match — e.g. Status is On loan.
containsText fields — Name contains laptop.
> < ≥ ≤Numbers and dates — Due back < today.
is empty / is not emptyWhether a field has any value.
Example: what's overdue

Filter Status is On loan and Due back < today to see every item that should already be back.

Sorting

Click any column header to sort by it; click again to reverse. An arrow shows the active sort. The automatic Inventory # column sorts numerically (1, 2, 3 …).

Choosing columns

Open ⋯ More → Columns to pick exactly which fields show in the table — keep it lean, or show everything. Your choice is remembered and can be saved into a view.

Inline editing

Double-click a cell to edit it right in the table — handy for a quick fix to a status, a date, or a Yes/No. Enter or click away to save, Esc to cancel.

  • Works for text, number, date/time, single-select and Yes/No cells (a dashed outline appears on hover).
  • Complex fields (relations, files, multi-select) and read-only auto/computed fields open the full editor instead.
  • A single click still opens the record's detail — a short pause tells a click from a double-click.

Working with records

Adding a record

  1. Click New record. If the register has data-entry forms, choose one — otherwise you get all fields (“Blank”).
  2. Fill in the fields. Required fields are marked; the form's rules may show, hide or require fields as you type.
  3. For Category, search and pick a record from the linked Categories register. For Assigned to, search a user.
  4. Click Save. The new row appears in the table.
Live rules

In our example, a rule makes Assigned to and Due back required the moment you set Status to On loan — so a loan is never recorded without a borrower and a return date.

Record detail & history

Click a row to open its detail — every field, including relations and attached files, read-only. At the bottom, a collapsible History timeline shows who created or changed the record, when, and what changed.

Editing & deleting

Open a record and choose Edit, or use the row's edit action. To remove one, use Delete (you'll be asked to confirm).

Who can change what

With Write access you can add records and edit or delete the ones you created. A register Manager can edit or delete any record. These rules are enforced on the server, not just hidden in the UI.

Importing & exporting CSV

Under ⋯ More:

  • Export CSV downloads exactly what's in the current (filtered/sorted) table — open it in any spreadsheet.
  • Import CSV uploads rows; columns are matched to fields by their header. A help dialog explains the format, offers a header-only template, and reports any per-row errors.
Bulk import is quiet

Importing many rows does not fire per-row automations — so a one-off load of 500 items won't send 500 notifications.

Saved views

Once you've set up a useful slice of the table — say, “items currently on loan, soonest return first” — save it so you (and your team) can come back to it in one click.

  1. Set the columns, filter, sort and search you want.
  2. Open ⋯ More → Save current view, give it a name (e.g. On loan).
  3. Choose private (just you) or shared (everyone who can see the register).
  4. Switch between saved views from the selector in the toolbar.

Sharing & access

A register's owner (and any manager) can share it with specific users or groups, each at one of three levels:

LevelCan…
ReadBrowse records and views.
WriteRead, plus add records and edit/delete their own.
ManageEverything, plus design the register: fields, forms, rules, automations and sharing.
  1. Open the register's Share dialog.
  2. Search a user or group (matches the id and the display name).
  3. Pick a role and Add. Remove access from the same dialog.

Building the register managers

If you can Manage a register, three extra tabs let you shape it. This is a quick tour — each has its own detailed guide.

Fields

Define the schema: add fields, pick a type (text, number, date, select, user, relation, file, computed, automatic…), set options, and mark required/unique. Drag to reorder.

Forms

A drag-and-drop builder: drag fields into sections, reorder, and preview each control. Build focused data-entry forms (e.g. a short “Log a loan” form).

Rules

Conditional logic that runs live as a form is filled and is re-checked on the server: show/hide a field, require-if, set a default, validate, or compute a value. One rule definition, two engines that agree.

Automations

Automations make a register do things. On a record create / update / delete, when an optional condition holds, DataForms runs an action on the server. There are nine actions — from a simple notification to creating folders, calendar events, Talk rooms and Deck boards.

ActionDoes
Send a notificationA Nextcloud notification to chosen people.
Send an emailAn email to chosen users.
Set a fieldSets a field on the record (e.g. stamp a status).
Create foldersA folder tree in the record owner's Files.
Copy a templateCopies template files into a folder.
Add a calendar eventAn event from a date field.
Create a Talk roomA conversation with participants + a welcome message.
Create a Deck boardA board with columns.
Call a webhookPOSTs the record to an external URL (SSRF-guarded, optionally signed).
Example for our register

“When a record is updated and Status becomes On loan, notify the person in Assigned to.” Set the trigger to update, the condition to Status is On loan, and the action to Send a notification.

Some actions (Talk room, Deck board) need an administrator to set up a service account first. See the written Automations guide and Admin & integration guide for the full detail.

FAQ

The table looks empty after I filtered — did I lose data?

No. A filter only hides non-matching rows. Clear the filter (and the search box) to see everything again. The record count next to the toolbar tells you how many match.

Why can't I edit some records?

With Write access you can edit only records you created. Ask a register Manager if you need to change someone else's record, or to be given Manage rights.

Why don't I see the Fields / Forms / Rules tabs?

Those are manager-only. If you only add and edit records, you'll see just the Records tab — by design, to keep things uncluttered.

Can I undo a delete?

Deletes are confirmed first. Records are soft-deleted server-side, but there's no in-app “restore” yet — so delete deliberately, and rely on your Nextcloud backups for recovery.

How do I link to a specific register?

The URL updates as you navigate (it reflects the open register and tab). Use the Copy link button in the header to share a deep link that opens straight to that register.
